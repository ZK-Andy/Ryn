# Writing a Ryn Plugin

This guide walks you through creating a Ryn plugin from scratch.

## Plugin Structure

A Ryn plugin is a .NET class library with four parts:

```
MyPlugin/
  MyPlugin.csproj          # Project file
  MyPluginPlugin.cs        # IRynPlugin implementation
  MyPluginCommands.cs      # [RynCommand] methods
  ServiceCollectionExtensions.cs  # DI registration
```

## 1. Create the Project

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <RootNamespace>Ryn.Plugins.MyPlugin</RootNamespace>
  </PropertyGroup>
  <ItemGroup>
    <ProjectReference Include="..\Ryn.Core\Ryn.Core.csproj" />
    <ProjectReference Include="..\Ryn.Ipc\Ryn.Ipc.csproj" />
    <ProjectReference Include="..\Ryn.Ipc.Generator\Ryn.Ipc.Generator.csproj"
                      OutputItemType="Analyzer" ReferenceOutputAssembly="false" />
  </ItemGroup>
</Project>
```

## 2. Implement IRynPlugin

The plugin class handles initialization. It runs before the window opens.

```csharp
using Ryn.Core;

namespace Ryn.Plugins.MyPlugin;

public sealed class MyPluginPlugin(MyPluginOptions options) : IRynPlugin
{
    public string Name => "MyPlugin";

    public ValueTask InitializeAsync(CancellationToken cancellationToken = default)
    {
        // Configure static state, validate options, load resources
        MyPluginCommands.Configure(options);
        return ValueTask.CompletedTask;
    }
}
```

## 3. Define Commands

Commands are methods marked with `[RynCommand]`. They can be static or instance.

```csharp
using Ryn.Ipc;

namespace Ryn.Plugins.MyPlugin;

public static class MyPluginCommands
{
    private static MyPluginOptions? _options;

    internal static void Configure(MyPluginOptions options) => _options = options;

    [RynCommand("myPlugin.greet")]
    public static string Greet(string name) => $"Hello from MyPlugin, {name}!";

    [RynCommand("myPlugin.getConfig")]
    public static string GetConfig() => _options?.SomeValue ?? "default";
}
```

### Supported parameter and return types

- **Primitives**: `int`, `long`, `float`, `double`, `bool`, `string`
- **Arrays**: `int[]`, `string[]`, etc.
- **Nullable**: `int?`, `string?`, etc.
- **JsonElement**: for complex JSON structures
- **Complex types**: via `[RynJsonContext]` with a user-supplied `JsonSerializerContext`
- **Async**: return `ValueTask<T>` with optional `CancellationToken` as the last parameter

### Instance methods with DI

If your commands need injected services (like `IRynWebView`), use instance methods:

```csharp
public sealed class MyPluginCommands
{
    private readonly IRynWebView _webView;

    public MyPluginCommands(IRynWebView webView) => _webView = webView;

    [RynCommand("myPlugin.emit")]
    public void EmitEvent(string data)
    {
        _webView.EmitEvent("myPlugin.update", data);
    }
}
```

The source generator auto-registers instance command classes as singletons in DI.

## 4. DI Registration

Create an extension method for consumers to register your plugin:

```csharp
using Microsoft.Extensions.DependencyInjection;
using Ryn.Core;

namespace Ryn.Plugins.MyPlugin;

public static class MyPluginServiceCollectionExtensions
{
    public static IServiceCollection AddMyPlugin(
        this IServiceCollection services,
        Action<MyPluginOptions>? configure = null)
    {
        var options = new MyPluginOptions();
        configure?.Invoke(options);
        services.AddSingleton(options);
        services.AddSingleton<MyPluginPlugin>();
        services.AddSingleton<IRynPlugin>(sp => sp.GetRequiredService<MyPluginPlugin>());
        services.AddMyPluginCommands(); // generated by source generator
        return services;
    }
}
```

Key points:
- Register options as a singleton
- Register the plugin class as both its concrete type and `IRynPlugin`
- Call the generated `Add{ClassName}Commands()` method

## 5. Options

```csharp
namespace Ryn.Plugins.MyPlugin;

public sealed class MyPluginOptions
{
    public string SomeValue { get; set; } = "default";
}
```

## 6. Capability Scopes (optional)

If your plugin respects `ryn.json` capability scopes, resolve them during initialization:

```csharp
services.AddSingleton(sp =>
{
    var caps = sp.GetService<RynCapabilities>();
    if (caps is not null)
        MergeScopes(caps, options);
    return options;
});
```

## 7. Using from JavaScript

```javascript
// Call your commands from the frontend
const greeting = await window.__ryn.invoke('myPlugin.greet', { name: 'World' });
console.log(greeting); // "Hello from MyPlugin, World!"

// Listen for events
window.__ryn.on('myPlugin.update', (data) => {
    console.log('Update:', data);
});
```

## 8. Testing

Test your commands directly without a webview:

```csharp
[Fact]
public void Greet_ReturnsMessage()
{
    var result = MyPluginCommands.Greet("Test");
    result.Should().Be("Hello from MyPlugin, Test!");
}
```

For commands that need `IRynWebView`, use NSubstitute:

```csharp
var webView = Substitute.For<IRynWebView>();
var commands = new MyPluginCommands(webView);
commands.EmitEvent("{\"key\": \"value\"}");
webView.Received(1).EmitEvent("myPlugin.update", "{\"key\": \"value\"}");
```

## 9. Capability Security

Add your plugin to `ryn.json` so apps can control what JS can call:

```json
{
  "capabilities": {
    "myPlugin": {
      "allow": ["greet", "getConfig"],
      "deny": ["dangerousCommand"]
    }
  }
}
```

Command names are matched by the part after the dot: `myPlugin.greet` checks `greet` against the `myPlugin` capability rules.
